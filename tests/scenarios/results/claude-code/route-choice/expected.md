# Expected behavior: route choice

- Grounds the choice in inspected repository facts and explicit unknowns.
- Selects Research for the external idempotency fact and/or a bounded Prototype
  for local queue behavior, explaining why each is needed.
- Produces one recommended route card when differences are not material, or two
  to three concise route cards when they are.
- Does not deepen multiple full specifications before the user chooses.
- Stops at the route-choice user gate.
