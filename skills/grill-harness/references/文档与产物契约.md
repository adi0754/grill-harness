# 文档与产物契约

真实仓库、决策账本、已批准规格、任务图与任务包、测试证据、`state.yaml` 依次承担不同权威；索引不得覆盖原始产物。

稳定 ID 为 `REQ-001`、`DEC-001`、`CON-001`、`RISK-001`、`CHG-001`、`TASK-001`、`ISSUE-001`、`EVD-001`。修改记录增加版本并保留替代关系，不静默覆盖。机器文件名和字段遵循现有脚本协议。

所有角色任务必须给出本地项目绝对路径、Git 基线、输入产物版本、允许/禁止范围、停止条件和报告绝对输出路径。证据记录实际命令、工作目录、时间、退出码、原始输出路径、基线、产生者和可复现性；无有效证据不得宣称完成。

长期文档不保存完整对话、临时推理或整段日志，只链接原始证据。
