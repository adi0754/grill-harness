# 文档与产物契约

## 唯一写入边界

所有 Grill Harness 状态、提示词、任务包、报告和知识文档只能写入当前工作流目录：

```text
~/.grill-harness/项目/<项目简称>-<项目短ID>/工作流/<日期>-<需求简称>-<工作流短ID>/
```

除已通过最终规格门禁且任务包明确授权的产品实现外，目标仓库只读。禁止把 Harness 指针、`.grill-harness`、需求文档、路线卡、任务包或报告写进产品仓库。`~/.codex`、`~/.claude` 等 Agent 目录也不是运行时产物目录。

## 当前工作流结构

```text
核心文档/
  需求基线.md
  决策账本.yaml
  领域词汇.md
  当前规格.md
  任务图.yaml
过程产物/
  需求审问/
  路线评估/
  研究与原型/
  仓库挑战/
  任务交接/
  实施报告/
  审查修复/
最终产物/
  最终规格.md
  集成报告.md
  验收报告.md
  项目经验.md
系统/
  state.yaml
  artifacts.yaml
  tasks.yaml
  evidence.yaml
```

- **核心文档**：随当前决策维护的权威工作合同；旧版本通过系统索引保留。
- **过程产物**：可审计的访谈、候选、调查、任务包和角色报告，不代表最终结论。
- **最终产物**：用户批准的唯一规格、集成和验收结论、可复用经验。
- **系统**：机器协议与索引，不覆盖人读产物，也不代替真实仓库事实。

模板来自 Skill 的 `assets/templates/`，实例必须复制到上述对应目录。模板中的输出路径一律改为当前工作流下的绝对路径。

## 权威顺序与稳定 ID

真实仓库说明当前实现事实；决策账本说明用户确认的意图；已批准最终规格说明实施合同；任务图与任务包说明局部授权；证据说明是否实现；系统文件只说明索引和流程位置。冲突时不按文件新旧或内容完整度自动选边。

稳定 ID：`REQ-001`、`DEC-001`、`CON-001`、`RISK-001`、`CHG-001`、`TASK-001`、`ISSUE-001`、`EVD-001`。每条记录包含版本、状态、来源、依赖/覆盖关系和替代项。修改增加版本并保留旧版本，禁止静默覆盖。

## 产物最低契约

| 产物 | 必需内容 |
|---|---|
| 需求基线 | 目标、非目标、场景、需求/约束、边界、验收信号、未决项、门禁版本 |
| 领域词汇 | 规范术语、定义、反例/易混词、领域关系；不含实现细节 |
| 路线卡 | 核心思路、边界、决策组合、优劣、风险、验证需要、推荐度 |
| 研究/原型 | 问题、方法、原始证据、结论、限制、对决策的影响 |
| 仓库挑战 | 被挑战假设、真实路径/符号/调用链、证据、结论、规格影响 |
| 最终规格 | ID 映射、仓库事实、接口/数据/错误/兼容/迁移、测试、验收、风险 |
| 任务图 | DAG、阻塞边、Frontier、写路径、共享契约、并行判定、Git 基线 |
| 角色任务包 | 角色目标、绝对路径、输入版本、授权/禁止范围、停止条件、输出契约 |
| 角色报告 | 实际读取/修改、完整 diff、命令证据、ID 覆盖、问题和剩余风险 |
| 验收报告 | 对当前基线的逐条验收、证据、结论和发布阻塞项 |

## 证据与长期知识

证据记录实际命令、绝对工作目录、开始/结束时间、退出码、Git 基线、产生者、原始输出绝对路径和可复现性。摘要可以链接证据，不能替代证据。证据与当前基线不符即失效。

长期文档不保存完整聊天、临时推理、凭据、环境变量值或整段日志。项目经验只保留稳定决策、否决路线及原因、领域知识、重要仓库事实、踩坑、证据索引、剩余风险和下次建议。
